## 3. MVP機能スコープ

### 含む機能
1. **作品登録** — 画像アップロード（〜数十点）
2. **定期巡回・類似検知** — AI改変・トリミング・反転にも耐える類似検知
3. **アラート** — スクリーンショット＋URL＋取得日時＋視覚比較つき通知
4. **削除要請テンプレート自動生成** — プラットフォーム別・日本語テンプレ
5. **案件管理** — 発見 → 通報済み → 対応中 → 解決 のステータス管理

### 含まない機能（MVP外）
- 回収代行・法的手続き代行（弁護士費用・非弁リスクのため）
- 英語・中国語テンプレ（フェーズ2以降）
- 海外EC（SHEIN/Temu）対応（フェーズ3以降）

## 4. 技術スタック（予定）

### フロントエンド
- 単一HTMLファイル（LP）
- 本番：React / Next.js（TBD）

### バックエンド
- 画像埋め込み: **CLIPモデル**（AI改変に耐える類似検知）
- ベクトルDB: **Qdrant**（近傍検索）
- クローラー: Python（対象プラットフォームのAPI/公開検索を優先）
- API: FastAPI or Node.js（TBD）

### インフラ
- クラウド: TBD（AWS / GCP / Fly.io 等）
- ストレージ: S3互換
- デプロイ: Vercel（フロント）+ コンテナ（バックエンド）

### なぜCLIP＋Qdrantか
商用API（TinEye API ≈ $200/月/5,000件）を全ユーザー全画像に使うと原価が利益を上回る。CLIPによる自前埋め込み＋ベクトルDB近傍検索が原価最適解。

## 7. 法務リスクと対策

### リスク1：クローリングの適法性
- **対策**: 公式API・RSS・公開検索等「許可された取得手段」を優先。
- robots.txt / 各社利用規約を対象ごとに事前精査し、可否一覧を管理。
- **弁護士スポット相談を初期コストとして計上**（着手前に必ず実施）。

### リスク2：非弁行為
- **境界線**: テンプレート提供 = OK / 代理送付・交渉 = NG
- 生成するのは「削除要請のひな形」にとどめ、ユーザー自身が送付する設計。
- 弁護士確認済みの文面範囲を文書化してサービスに明示。

## 9. GTM（Go-To-Market）戦略

### 獲得チャネル
- pixiv・X上のイラストレーターコミュニティへの直接アプローチ
- minne/Creema作家コミュニティ、ハンドメイド系ブロガー/YouTuber
- 被害体験を発信している作家をアーリーアダプター兼広報に

### 無料フック
「無料スキャン体験」— 作品画像を受け取り、運営者が手動でチェックして被害レポートを返す（MVP実装前はコンシェルジュ型）。

### PR文脈
SHEIN/Temu模倣問題は報道フックが強い。初期はプレスリリース不要、作家コミュニティでの口コミを優先。
