# リランキング効果の実測所見（フェーズ2）

> **位置づけ**: これは「pHashで足りる/CLIP必須」の**最終判定ではない**。独立監査の指摘に従い、
> **方向性シグナル**（どの加工で安価ハッシュが破綻するか）と、**本番評価の仕様**（判定に格上げする条件）を示す。
> 実データは surrogate（ローカル実画像21枚）であり、実際のターゲットPF加工リポストではない。

## 実験設計（監査反映済み）

- **手法**: 本物の pHash（`imagehash`/DCT・256bit）と、対照の aHash（`rerank.py` の16×16平均ハッシュ）を併走。
  - 監査の致命指摘「`imagehash`未導入だと"pHash"が実は aHash」を回避するため、**本物 pHash を必須化**し明示ラベルで併記。
- **分離性を測る**: 陽性のみの自己類似ではなく、各 original に対しコーパスの他画像を **distractor** として併置。
  - **主指標 `retrieval_rate`** = 加工版（陽性）が全 distractor を上回った original の割合（＝加工しても元を取り違えず引ける率）。
- **尺度**: 正規化ハミング類似度（pHash/aHash は同尺度で相互比較可）。CLIP は尺度が異なるため本実験では扱わない。
- 再現: `python3 poc/eval_transforms.py`（出力 `poc/out/transform_eval.md` / `.csv`）。torch 不要。

## 結果（コーパス21枚・retrieval_rate）

| 加工の種類 | 例 | pHash | aHash | 解釈 |
|---|---|---|---|---|
| **測光系** | jpeg圧縮・明度・コントラスト・彩度・**色相**・グレースケール・ぼかし・リサイズ・小クロップ・小回転・遮蔽 | 0.90 | 0.90 | **安価ハッシュで回収可** |
| **幾何系** | **左右反転** | 0.00 | 0.19 | 両方**破綻** |
| | **15°回転** | 0.14 | 0.57 | pHash破綻・aHashも不十分 |
| | **25%クロップ** | 0.10 | 0.48 | 両方不十分 |
| | **コラージュ** | 0.05 | 0.14 | 両方破綻 |

（0.90 は天井：コーパスに同一書籍の別版など near-dup が2件混入し、identity でも 0.90 に頭打ち。§仕様参照）

## 所見（正直な非自明結果）

1. **測光系の加工は安価な pHash/aHash で十分**。色相シフト60°やグレースケールでも retrieval 0.90 を維持。
   → SPEC が「フェーズ2の課題」とする色変え系の相当部分は、**CLIP を持ち出さずに回収できる**見込み。

2. **幾何系（反転・大回転・大クロップ・コラージュ）は両ハッシュとも破綻**（margin が負＝distractor に埋没）。
   → ここが埋め込み（CLIP/DINO）または**加工前正規化**（反転・回転のマルチハッシュ）の出番。**次段で検証すべき唯一の領域**。

3. **pHash は aHash を明確に上回らなかった（差 ≥0.2 の加工は0件）。むしろ幾何破綻ケースでは aHash の方がやや頑健**
   （反転 0.19>0.00、15°回転 0.57>0.14、25%クロップ 0.48>0.10）。DCT の pHash は幾何ズレで低周波成分が動くため、
   空間平均の aHash より脆いことがある。
   → 監査が懸念した「aHash を pHash と誤ラベル→CLIP必須の誤結論」は、**このデータでは結論を変えない**
   （安価ハッシュ全般が幾何系で破綻、という同じ結論）。ただし本物 pHash で測ったので**正しくそう主張できる**。

## 暫定的な方向性（判定ではない）

- **測光系のみが問題なら、pHash（またはaHash）ベースの二次リランキングで足りる可能性が高い** → 重い CLIP/GPU/Qdrant 投資は不要。
- **反転・大回転・コラージュ耐性が要件なら、安価ハッシュでは不足**。ただし選択肢は CLIP だけではない：
  - (a) **加工前正規化**（反転・複数回転のマルチハッシュ）＝ 依然 torch 不要・低コスト。
  - (b) CLIP/DINO 埋め込み ＝ 高コスト（別コンテナ・GPU）。**(a) を先に試してから (b) を検討**するのが監査整合。

## 本番評価の仕様（この結果を「判定」に格上げする条件）

1. **実 PF 陽性データ**：メルカリ/pixiv 等で実際に加工転載された作品と原画のペア（現状の保存レスポンスには無い＝前回監査の確定事項）。
2. **標本下限**：`eval_rerank.py` の PROVISIONAL 閾値（クエリ ≥5・陽性 ≥20）以上。
3. **distractor 統制**：本コーパスの near-dup 混入（0.90天井の原因）を除くか別クラス化する。
4. **CLIP を比較する場合**：生スコアは尺度非互換のため比較不可。**ランク指標**（Recall@k/mAP、`eval_rerank.py`）または**手法別較正**（各手法の null 分布に対する z 値）で行う。
5. **幾何系の次段実験**：反転・回転・クロップ・コラージュに絞り、(a) マルチハッシュ正規化 →（不足なら）(b) CLIP/DINO、の順で distractor 付き retrieval を測る。
